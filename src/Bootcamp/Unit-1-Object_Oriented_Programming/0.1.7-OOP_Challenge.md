# Object Oriented Programming Challenge - Scooter Hire System

## Learning Objectives
- Demonstrate what you have learnt in this Unit by creating UML diagrams, classes and unit tests for a new domain.

## Pre-requisites

- Completion of this unit's lessons

## Lesson

You have been asked to design an Electric Scooter Hire app system. Listen to this video to find out how Electric Scooters work in London:

<iframe width="560" height="315" src="https://www.youtube.com/embed/JY5EB8Qp1L8" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>

You may be wondering how the scooters get charged? The scooter company employs scooter "juicers", to collect scooters from the streets after 9:00pm and charge them at their homes. 

Here is a summary of the Scooter Hire requirements:
  * To hire a scooter, a user must download the Electric Scooter Hire app
  * Prior to hire a user must register their personal details, payment details and upload their driving license
  * The app displays the location of available scooters
  * To commence hire, a user scans the scooter QR code
  * Scooters can only operate within a restricted location (geofencing)
  * Scooter are limited to a speed of 20 km/h when fully charged and 12 km/h when the battery is low
  * The user is alerted to a low battery
  * As part of ending the ride, the user is asked to take a photo of the scooter
  * Payment is taken once the ride is ended
  * It takes approximately 3 hours to charge a scooter


## Assignment
To complete the assignment you must create the following:

1. Design your solution based on the above requirements by creating the following UML diagrams:
   * A UML Use Case diagram showing the interactions between the User Actor, Scooter Actor and the Electric Scooter Hire app system. Also include Use Cases showing the interaction between the Juicer Actor and Scooter Actor
   * A UML Class diagram showing the system's classes, their attributes, methods and inter-relationships. Hint: look out for the "nouns" in the requirements / when watching the video, classes are always nouns.
   * A UML Sequence diagram showing the interactions between the user Actor, the Scooter Actor and the classes in the system

   Screenshots of your UML diagrams must be committed to your GitHub repository. You should get the diagrams checked by your coach before starting to code.

1. Implement your system as follows:
   - Create a [new Node.js project](/curriculum/Bootcamp/FAQ#createNewProject) for this assignment.
   - A `/src` directory containing classes with interactions (methods calls) that capture the required functionality
   - A `/tests` directory containing unit tests which test all scenarios
   - A screenshot of your [unit test coverage report](/curriculum/Bootcamp/FAQ#generateCoverage) (aim for 100% coverage)

   Commit your code into Github and share the link with your coach for review.

### Assignment extension tasks

Use the [PlantUML documentation](https://plantuml.com/sequence-diagram) `alt/else` syntax to extend your UML Sequence diagrams to include the handling of error conditions such as the scooter breaking when the user has started the hire period.
