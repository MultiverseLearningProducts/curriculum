## Lesson
Objects are the main data type in Javascript. Think of objects as nouns (person, place or thing), something you can touch, see, smell, feel or taste.

Let's start by looking at the three different ways we can make an object:

```javascript
const person1 = {}
const person2 = new Object()
const person3 = Object.create({})
```
In this lesson we will focus on the first method.

Once you have an object you can add properties to it. Let's create a person and give them a `name` property:

```javascript
const person1 = {}
person1.name = 'Bosola'
person1.age = 21
```
Very easy, we just use the 'dot' notation and `=` operator to assign the property and a value. 

The above example could also be written using an object initializer, a comma-delimited list of property name/value pairs, enclosed in curly braces ({}):
```javascript
const person1 = {
  name: 'Bosola',
  age: 21
}
```

We can read the value from the property in a couple of different ways:
```javascript
person1.name
person1['name']
```
Both do the same thing. The reason for using the bracket notation is when our property is a string that has an unusual character in it i.e. "full-name". For example, this will not work:
```javascript
person1.full-name = 'Bosola Randle'
```
but this will
```javascript
person1['full-name'] = 'Bosola Randle'
```

Objects can reference other objects. For example, if we create objects to represent Bosola's parents, we can reference them within a array property. 

```javascript
const person2 = {
  name: 'Emile',
  age: 56,
  parents: []
}
const person3 = {
  name: 'Parul',
  age: 58,
  parents: []
}
const person1 = {
  name: 'Bosola',
  age: 21,
  parents: [person2, person3]
}
```

How could we print the name of each of Bosola's parents?

We can add a new property called `childOf` which executes a function to return the names of that object's parents. To access this object's parents array we use the `this` keyword:

```javascript
childOf: function () {
  return this.parents.map(parent => parent.name).join(' & ') || "unknown"
}
```
In the function above we use the Javascript array method [map](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/map) to pick out the name properties from each of the parent objects. We then [join](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/join) the names with together with an '&' in between them. If the array is empty, `join` will return an empty string which evaluates as falsey, so we return a default value.

Finally, since we want all our person objects to have this function, we can declare the function once by assigning it to a variable. Now our `childOf` property can have as its value a reference to the `childOf` function. (Note that you must use the `function` keyword for this to work. Arrow functions reference the `this` of the parent's scope so do not work the same way).
